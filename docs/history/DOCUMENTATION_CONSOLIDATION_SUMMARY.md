## Maintenance Guidelines

### Adding New Documentation

1. **Determine topic** - Setup, Usage, Features, Troubleshooting?
2. **Check existing docs** - Can it be added to existing doc?
3. **Create or update** - New doc or section in existing?
4. **Update INDEX.md** - Add to master index
5. **Cross-reference** - Link from related docs

### Updating Documentation

1. **Update content** - In appropriate doc file
2. **Update date** - "Last Updated" timestamp
3. **Check links** - Verify cross-references work
4. **Test examples** - Ensure code samples work

### Archiving Documentation

1. **Move to history/** - Don't delete
2. **Update history/README.md** - Document why archived
3. **Add migration note** - Point to new location
4. **Update references** - Fix any dead links
